> **版本**: V1.0

## 📌 框架总览

### 核心理念

```
精准匹配 = Σ(通用维度权重 × 特异化维度权重 × 实时动态系数)

护城河 = 时间积累 × 双边网络效应 × 数据飞轮
```

### 数据流向

```
C端AGENT (需求解码) ←→ 事实向量匹配引擎 ←→ B端AGENT (供给表达)
        ↓                                           ↓
   用户画像空间                                酒店特征空间
   出行意图空间                                服务能力空间
   偏好指纹空间                                场景适配空间
   行为轨迹空间                                内容语义空间
   实时动态空间                                动态运营空间
   对话语义空间                                关系网络空间
```

| 服务期待 | 口碑/评分 | 阈值过滤 | 0.07 |

### 动态权重调整规则

```
用户类型调整:
- 商务出行 → 场景权重+10%, 实时价格权重+5%
- 休闲度假 → 内容权重+10%, 场景权重+5%
- 家庭亲子 → 设施权重+10%, 口碑权重+5%

上下文调整:
- 明确特定需求 → 通用权重-10%, 特异化权重+10%
- 紧急预订 → 实时动态权重+15%
- 长停留时长 → 长住场景权重+10%
```
