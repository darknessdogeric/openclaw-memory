# HAL项目测试Query集

> **用途**: DeepSeek vs Qwen 对比测试  
> **场景**: AI民宿预订助手 (AHL-Dali)  
> **设计原则**: 覆盖典型用户意图，测试推理能力和领域理解  
> **生成时间**: 2026年2月27日  
> **版本**: v1.0

---

## 一、Query设计原则

### 1.1 覆盖维度

| 维度 | 说明 | Query数量 |
|------|------|-----------|
| **意图清晰度** | 模糊 vs 明确 | 各5个 |
| **复杂度** | 简单 vs 多条件组合 | 各5个 |
| **场景类型** | 情侣/家庭/商务/独行/团建 | 各2个 |
| **决策阶段** | 灵感收集 vs 具体预订 | 各5个 |
| **语言风格** | 口语化 vs 结构化 | 各5个 |

### 1.2 测试目标

1. **意图识别**: 能否准确理解用户需求
2. **实体提取**: 能否提取地点、时间、人数、偏好
3. **推理能力**: 能否处理模糊需求和隐性偏好
4. **领域知识**: 能否理解民宿行业术语和场景
5. **对话能力**: 能否进行多轮交互澄清

## 二、10个测试Query

### Query 1: 模糊情感型 (高难度)

**用户输入**: 
> "我想去大理放松一下，最好是有感觉的地方，不要那种太商业化的。大概住3天吧，预算无所谓，关键是要有那种...你懂的，那种氛围。"

**预期解析**:
- **意图**: 寻找有特色的民宿，强调体验而非硬件
- **地点**: 大理
- **时间**: 3天
- **人数**: 未明确 (需澄清)
- **偏好**: 非商业化、有氛围、有特色
- **预算**: 高预算 ("无所谓")
- **难点**: "有感觉"、"氛围"极其主观，需要推理用户可能想要的风格 (古城文艺? 海边宁静? 田园诗意?)

**评判标准**:
- [ ] 是否询问人数和入住日期?
- [ ] 是否提供几种"氛围"风格供选择?
- [ ] 是否主动询问具体偏好 (安静/热闹/海景/山景)?
- [ ] 推荐是否符合"非商业化"定位?

### Query 3: 复杂条件组合型 (高难度)

**预期解析**:

- **地点**: 大理，近洱海但不临水
- **时间**: 5天
- **人数**: 5人 (4大1小)
- **特殊需求**: 
  - 儿童友好 (院子、安全)
  - 老人友好 (暖气/空调)
  - 餐饮便利
- **预算**: ≤800元/晚
- **房型**: 需要家庭房或连通房

**评判标准**:
- [ ] 是否正确理解"4大1小"的房型需求?
- [ ] 是否推理出需要儿童安全设施?
- [ ] 是否理解"近洱海但不临水"的精确位置要求?
- [ ] 是否询问孩子性别/年龄以提供针对性服务?
- [ ] 推荐是否符合多代同堂需求?

### Query 9: 抱怨转需求型 (高难度)

**用户输入**:
> "上次去大理住的民宿太坑了，图片和实际完全不一样，晚上还吵得要死。这次我绝对不要那种网红店，就要实实在在的，干净、安静、舒服就行。价格好说，但别让我再踩雷了。你帮我严格把关，有问题我找你。"

**预期解析**:
- **意图**: 预订 (有负面经历后的谨慎)
- **地点**: 大理
- **时间**: 未明确
- **人数**: 未明确
- **负面经历**: 图文不符、噪音
- **核心需求**: 
  - 真实可靠 (非虚假宣传)
  - 安静 (隔音好)
  - 干净、舒适
  - 非网红店 (口碑导向)
- **预算**: 高 ("价格好说")
- **情绪**: 焦虑、不信任、需要 reassurance

**评判标准**:
- [ ] 是否表达共情和理解?
- [ ] 是否主动提供筛选标准 (如何避坑)?
- [ ] 是否询问上次具体遭遇以针对性推荐?
- [ ] 是否提供保障措施 (实拍图、住客评价、退改政策)?
- [ ] 是否建立信任 (而非过度承诺)?

### Query 10: 超长多需求型 (高难度)

**用户输入**:
> "我们是一个摄影协会的组织，大概15-20人去大理采风，时间大概是4月中旬，具体日期还没定。需要8-10间房，最好是在同一个院子或者相邻的民宿，方便集合。有几个老师年纪比较大了，不能爬太多楼梯，最好是一楼或者有电梯。另外我们需要一个公共空间可以放器材、晚上可以点评作品。早餐要早，因为我们要赶日出。还有，我们可能要提前寄一些器材过去，民宿能代收吗? 哦对了，其中有几个人是素食的，早餐要有素食选项。预算的话，每间房每晚控制在400以内吧。"

**预期解析**:
- **意图**: 摄影团建，专业需求
- **地点**: 大理
- **时间**: 4月中旬 (具体未定)
- **人数**: 15-20人，需要8-10间房
- **特殊需求** (非常多):
  - 团队住宿 (同院子/相邻)
  - 老年友好 (一楼/电梯，少楼梯)
  - 公共空间 (器材存放、作品点评)
  - 早餐早 (日出前)
  - 代收快递 (器材)
  - 素食选项
- **预算**: ≤400元/晚/间
- **组织性质**: 摄影协会 (可能有后续合作)

**评判标准**:
- [ ] 是否正确梳理所有需求 (清单式确认)?
- [ ] 是否确认团队结构 (年龄分布、房间分配)?
- [ ] 是否询问具体日期范围?
- [ ] 是否确认公共空间的具体要求 (投影、桌子)?
- [ ] 是否询问是否有后续合作可能 (B2B角度)?
- [ ] 推荐是否符合所有硬性约束?

### 3.3 评判维度详解

**意图识别 (Intent Recognition)**:
- 5分: 完全理解用户意图，包括隐性需求
- 3分: 理解主要意图，遗漏部分细节
- 1分: 误解用户意图

**实体提取 (Entity Extraction)**:
- 5分: 提取所有关键信息 (地点、时间、人数、偏好、预算)
- 3分: 提取主要信息，遗漏部分
- 1分: 遗漏关键信息

**推理能力 (Reasoning)**:
- 5分: 能处理模糊需求，进行合理推断
- 3分: 能处理部分模糊需求
- 1分: 只能处理明确指令

**领域知识 (Domain Knowledge)**:
- 5分: 准确理解民宿行业术语和场景
- 3分: 基本理解，部分术语有误
- 1分: 明显缺乏行业知识

**对话能力 (Conversation)**:
- 5分: 自然流畅，主动澄清，有同理心
- 3分: 能完成对话，但略显机械
- 1分: 生硬，需要用户反复解释

## 四、预期结果分析

### 4.1 DeepSeek预期优势

**Q1 (模糊情感型)**: 推理能力强，能更好理解"氛围"等抽象概念  
**Q3 (复杂条件组合)**: 逻辑推理强，能处理多代同堂的复杂需求  
**Q6 (情感叙事型)**: 理解隐性需求，提供情感化回应  
**Q9 (抱怨转需求型)**: 共情能力强，能有效处理负面情绪  

### 4.2 Qwen预期优势

**Q5 (专业术语型)**: 可能有更好的B2B领域知识  
**Q7 (极简明确型)**: 快速响应，处理简单查询效率高  
**Q10 (超长多需求型)**: 长文本理解能力，能梳理复杂需求清单  

### 4.3 平手预期

**Q2 (明确商务型)**: 两者都能很好处理  
**Q4 (口语化碎片化)**: 取决于对话能力  
**Q8 (多目的地对比)**: 取决于知识广度

## 五、决策阈值

```
评分结果处理:

如果 DeepSeek总分 - Qwen总分 ≥ 10分:
  → 明确选择DeepSeek

如果 0 < DeepSeek总分 - Qwen总分 < 10分:
  → DeepSeek略有优势，结合成本考虑选择DeepSeek

如果 Qwen总分 - DeepSeek总分 ≥ 5分:
  → 重新评估，可能特定场景需要Qwen

如果 特定Query类型Qwen明显更好:
  → 考虑混合方案 (主要用DeepSeek，特定场景调用Qwen)
```

## 六、使用建议

### 6.1 测试执行

1. **逐Query测试**: 不要一次发送所有Query，模拟真实对话流程
2. **记录完整对话**: 保存多轮交互过程，而非只看第一轮回复
3. **注意响应时间**: 记录每个Query的响应延迟
4. **截图保存**: 保存关键回复的截图用于对比

### 6.2 结果分析

1. **定量分析**: 使用评分表计算总分
2. **定性分析**: 记录主观感受 (流畅度、专业感、亲和力)
3. **成本对比**: 记录API调用成本 (Token消耗)
4. **综合决策**: 结合定量+定性+成本做出最终选择
