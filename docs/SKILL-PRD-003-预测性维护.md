## 1. 价值定位

### 1.1 解决的问题
酒店工程设备维护面临"三高三低"困境：

**高停机损失**
- 空调故障导致客房无法售卖，单日损失数万元
- 电梯困人事件引发宾客投诉和品牌损害
- 厨房设备故障影响餐饮营收

**高维修成本**
- 事后抢修费用是计划维护的3-5倍
- 紧急采购备件价格高昂且可能缺货
- 外包维修服务费时不菲

**高安全隐患**
- 设备带病运行可能引发安全事故
- 电气火灾、燃气泄漏等风险难以预知
- 合规检查压力日益增大

**低维护效率**
- 传统PPM(计划性维护)"一刀切"，过度维护与维护不足并存
- 工程师凭经验判断，知识难以传承
- 设备台账混乱，历史数据无法追溯

### 1.2 核心价值
**"从'事后救火'到'事前预防'，让设备拥有'健康管家'"**

- **提前预警**：故障发生前7-30天发出预警，从容安排维护
- **精准定位**：AI诊断故障类型和位置，减少排查时间
- **成本优化**：避免非计划停机，降低30-50%维修成本
- **寿命延长**：科学维护策略，延长设备使用寿命20%+

### 1.3 量化收益

| 指标 | 传统模式 | 预测性维护 | 提升幅度 |
|------|---------|-----------|---------|
非计划停机 | 12-15次/年 | 2-3次/年 | **降低80%** |
维修成本 | 100万元/年 | 60万元/年 | **节省40%** |
设备寿命 | 设计寿命 | 延长20% | **+20%** |
能耗成本 | 基准值 | -15% | **节省15%** |
安全事故 | 3-5起/年 | 0-1起/年 | **降低90%** |

#### 模块3: AI异常检测与故障预测

**异常检测算法**

```python
# 基于LSTM的时序异常检测
class AnomalyDetector:
    def __init__(self):
        self.lstm_model = load_model('lstm_anomaly.h5')
        self.threshold = 3.0  # 标准差倍数
    
    def detect(self, sensor_data):
        # 预测下一步正常值
        predicted = self.lstm_model.predict(sensor_data)
        # 计算预测误差
        error = abs(sensor_data[-1] - predicted)
        # 判断异常
        if error > self.threshold * std:
            return {
                'is_anomaly': True,
                'severity': self.calculate_severity(error),
                'confidence': self.calculate_confidence()
            }
```

**故障预测模型**

| 算法 | 适用场景 | 预测提前期 | 准确率 |
|------|---------|-----------|--------|
LSTM | 时序数据趋势预测 | 7-30天 | 85-90% |
Random Forest | 多维度特征分类 | 即时 | 92-95% |
SVM | 小样本异常识别 | 即时 | 88-92% |
**Ensemble** | 综合决策 | 7-30天 | **93-96%** |

### 4.2 核心算法

**振动分析算法**
```python
# 频谱分析识别轴承故障
def analyze_vibration(signal, fs=1000):
    # FFT变换
    fft = np.fft.fft(signal)
    freq = np.fft.fftfreq(len(signal), 1/fs)
    
    # 轴承特征频率
    BPFI = 5.415  # 内圈故障频率(转频倍数)
    BPFO = 3.585  # 外圈故障频率
    BSF = 2.357   # 滚动体故障频率
    
    # 检测特征频率峰值
    for characteristic_freq in [BPFI, BPFO, BSF]:
        if detect_peak(freq, fft, characteristic_freq):
            return f"轴承{get_location(characteristic_freq)}故障"
```

**寿命预测模型(RUL)**
```
输入: 设备运行历史数据
      ↓
特征工程:
  - 统计特征: 均值、方差、峰值
  - 频域特征: 频谱能量分布
  - 时域特征: 趋势、周期性
      ↓
LSTM网络:
  - 学习退化趋势
  - 预测剩余寿命
      ↓
输出: RUL = 预计剩余运行天数
```
