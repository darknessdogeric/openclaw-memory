## 三、【获客雷达中枢】6 SKILLs ★ 核心新增

### 定位: 7×24小时市场情报与获客机会监测

#### SKILL-RADAR-01 竞品价格雷达

**功能**: 全渠道竞品价格实时监测
**监测维度**:
```
【价格监测】
- 同商圈竞品酒店价格 (每小时抓取)
- 同档次酒店价格对比
- 竞品促销活动监测
- 竞品库存状态 (开房/关房)

【价格分析】
- 价格趋势分析 (7天/30天)
- 竞品定价策略识别
- 价格异常预警 (恶意低价)
- 最优价格定位建议
```

**预警机制**:
```
红灯预警: 竞品价格低于我方15%以上
黄灯提醒: 竞品推出大规模促销
蓝灯提示: 竞品价格上调，机会窗口
```

**输出**:
- 竞品价格日报
- 价格战风险预警
- 跟价/涨价建议

---

#### SKILL-RADAR-02 市场机会雷达

**功能**: 市场需求波动与机会识别
**监测指标**:
```
【需求信号】
- 搜索指数变化 (百度/微信指数)
- OTA搜索热度
- 展会/活动日历
- 节假日/学校假期

【供给信号】
- 竞品库存紧张度
- 商圈新开业酒店
- 竞品装修停业
- 大型团体预订

【价格信号】
- 竞品价格上调
- 市场均价上涨
- 尾房减少
```

**机会识别**:
```
机会类型A: 竞品满房 → 我方提价/截流
机会类型B: 大型活动 → 提前锁房/涨价
机会类型C: 搜索上涨 → 加大投放
机会类型D: 竞品差评 → 针对性营销
```

---

#### SKILL-RADAR-03 客户行为雷达

**功能**: 潜在客户行为追踪与意向识别
**追踪维度**:
```
【浏览行为】
- 官网/App浏览轨迹
- 搜索关键词记录
- 房型偏好分析
- 价格敏感度分析

【互动行为】
- 咨询未预订客户
- 收藏未下单客户
- 加购物车未支付
- 领取优惠券未使用

【流失预警】
- 历史客户流失预警
- 会员活跃度下降
- 协议客户消费下降
```

**意向评分模型**:
```
高意向 (90-100分): 浏览3次+咨询+加购物车
中意向 (60-89分): 浏览2次+收藏
低意向 (30-59分): 单次浏览+跳出率高
无意向 (0-29分): 仅首页浏览
```

---

#### SKILL-RADAR-04 渠道效能雷达

**功能**: 全渠道获客效率实时监测
**监测指标**:
```
【流量指标】
- 各渠道UV/PV
- 跳出率/停留时长
- 新客占比

【转化指标】
- 转化率 (流量→订单)
- 客单价
- ROI (投入产出比)
- CPA (获客成本)

【质量指标】
- 取消率
- No-Show率
- 好评率
- 复购率
```

**渠道健康度评分**:
```
健康度 = 流量得分×30% + 转化得分×40% + 质量得分×30%
```

**自动优化建议**:
```
低效渠道 → 减少投放预算
高效渠道 → 增加投放预算
异常波动 → 预警通知
```

---

#### SKILL-RADAR-05 舆情口碑雷达

**功能**: 全网口碑与舆情监测
**监测范围**:
```
【OTA平台】
- 携程/美团/飞猪点评实时监控
- 差评关键词提取
- 竞品好评分析 (学习借鉴)

【社交媒体】
- 小红书提及监测
- 抖音视频监测
- 微博话题监测
- 知乎问答监测

【新闻媒体】
- 行业媒体报道
- 地方新闻提及
- 危机舆情预警
```

**情感分析**:
```
正面情感 (>0.6): 加强传播
中性情感 (0.3-0.6): 引导转化
负面情感 (<0.3): 危机预警
```

**危机预警**:
```
一级预警: 负面信息24小时内传播量>1000
二级预警: 负面信息涉及安全/卫生
三级预警: 媒体跟进报道
```

---

#### SKILL-RADAR-06 预测性获客雷达

**功能**: AI预测未来获客机会与风险
**预测模型**:
```
【需求预测】
- 未来7/14/30天预订需求
- 节假日需求高峰
- 展会/会议需求

【竞争预测】
- 竞品促销概率
- 市场价格走势
- 竞品库存策略

【客户预测】
- 高意向客户识别
- 流失客户预测
- LTV潜力客户
```

**策略建议引擎**:
```
输入: 雷达监测数据 + 预测模型
    ↓
处理: AI策略建议生成
    ↓
输出: 
- 明日价格调整建议
- 本周促销策略建议
- 本月渠道预算分配建议
- 即时跟价/涨价建议
```

**自动化决策**:
```
场景1: 预测明日需求上涨20%
建议: 关闭低价房型，开启溢价策略

场景2: 竞品明日满房
建议: 截流策略，价格上调10%

场景3: 高意向客户未下单(3次浏览)
建议: 自动推送限时优惠券
```

## 十三、获客雷达与全体系协同

### 13.1 获客雷达数据流向

```
【获客雷达中枢】
    ├── 竞品价格雷达 → 定价策略优化
    ├── 市场机会雷达 → 促销活动策划
    ├── 客户行为雷达 → 个性化推荐
    ├── 渠道效能雷达 → 预算分配优化
    ├── 舆情口碑雷达 → 品牌策略调整
    └── 预测性雷达 → 前瞻性决策
    
    ↓ 数据汇聚
    
【各AGENT策略调整】
    ├── 品牌营销: 根据舆情调整传播策略
    ├── 数字商务: 根据渠道效能调整投放
    ├── 会员CRM: 根据客户行为精准营销
    ├── 销售: 根据市场机会主动出击
    └── 收益管理: 根据竞争情报动态定价
```

### 13.2 7×24小时自动化闭环

```
00:00-06:00: 夜间数据抓取与分析
06:00-09:00: 晨会数据报告生成
09:00-12:00: 策略建议推送
12:00-14:00: 午间价格监控
14:00-18:00: 下午策略优化
18:00-22:00: 晚间促销监控
22:00-24:00: 夜间模式调整
```
