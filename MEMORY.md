### OTA运营SKILL体系 V1.0
**创建**: 2026-03-01 (基于携程实地观察+全网知识提炼)  
**来源**: 携程平台实地研究 + 酒店行业知识库 + 收益管理最佳实践  
**SKILL数量**: 15个细分SKILL，覆盖8大运营模块  
**适用平台**: 携程/美团/飞猪/Booking/Agoda等全渠道

**核心架构**:
```
OTA运营成功公式 = 流量 × 转化率 × 客单价 × 复购率

流量获取: 挂牌等级 + 参与活动 + 关键词优化
转化率: 首图 + 价格 + 评价 + 详情页
客单价: 动态定价 + 促销设计 + 房型升级
复购率: 服务体验 + 会员体系 + 私域运营
```

**8大SKILL模块**:

| 模块 | SKILL数量 | 核心能力 | 关键指标 |
|------|-----------|----------|----------|
| **OTA-STR 渠道策略** | 2个 | 渠道组合设计/差异化定价 | 渠道ROI |
| **OTA-LIST 房源优化** | 3个 | 首图设计/标题优化/设施标签 | CTR/转化率 |
| **OTA-PRICE 定价促销** | 2个 | 动态定价/促销活动设计 | ADR/RevPAR |
| **OTA-REVIEW 评价运营** | 2个 | 好评引导/差评应对 | 好评率/DSR |
| **OTA-CONV 转化优化** | 2个 | 详情页优化/客服响应 | 转化率 |
| **OTA-BADGE 挂牌冲刺** | 1个 | 特牌争取策略 | 挂牌等级 |
| **OTA-DATA 数据分析** | 2个 | 数据日报/竞品分析 | 核心指标监控 |
| **OTA-OMNI 公私域联动** | 1个 | 公域转私域导流 | 导流率/复购率 |

**15个细分SKILL清单**:

**渠道策略模块 (OTA-STR)**:
- SKILL-001 渠道组合设计: 根据酒店定位设计OTA渠道组合(携程/美团/Booking等)
- SKILL-002 渠道差异化定价: 设计差异化价格体系，避免比价损失

**房源优化模块 (OTA-LIST)**:
- SKILL-003 高转化首图设计: CTR优化，A/B测试方案
- SKILL-004 标题关键词优化: 搜索排名优化，关键词库
- SKILL-005 设施标签策略: 25+标签完善，差异化标签配置

**定价促销模块 (OTA-PRICE)**:
- SKILL-006 动态定价策略: 需求系数×竞争系数×时间系数模型
- SKILL-007 促销活动设计: 早鸟/连住/秒杀/套餐/大促方案

**评价运营模块 (OTA-REVIEW)**:
- SKILL-008 好评引导体系: 邀评SOP，转化率目标30%
- SKILL-009 差评应对与修复: 分级响应，申诉策略

**转化优化模块 (OTA-CONV)**:
- SKILL-010 房源信息优化: 15-30张图，详情页优化清单
- SKILL-011 客户服务响应: <5分钟响应，话术库

**挂牌冲刺模块 (OTA-BADGE)**:
- SKILL-012 携程特牌争取: 金牌/钻石/金钻冲刺计划

**数据分析模块 (OTA-DATA)**:
- SKILL-013 OTA数据日报: 8大核心指标监控
- SKILL-014 竞品对标分析: 5维度竞品分析模板

**公私域联动模块 (OTA-OMNI)**:
- SKILL-015 公域转私域导流: 触点设计，诱饵设计，导流率目标30-50%

**携程平台核心特征**:
- 规模: 170万+酒店 | 200+国家 | 9万+城市
- 品类: 酒店/民宿/公寓/客栈/青旅/农家乐/别墅/特色住宿
- 保障: 放心住(确认不涨价) | 放心付 | 会员积分体系
- 排名算法权重: 销量25% + 转化率20% + 好评15% + 响应15% + 挂牌10% + 活动10% + 信息5%

**实战应用场景**:
1. 新酒店OTA冷启动 (90天计划: 基础搭建→流量获取→等级冲刺)
2. 存量酒店OTA优化 (数据诊断→短板补齐→排名提升)
3. 多平台渠道管理 (渠道组合→差异化定价→统一库存)
4. 收益最大化策略 (动态定价→促销组合→私域沉淀)

### AHL事实双向向量数据维度框架
**位置**: `memory/ahl-vector-data-framework.md`  
**版本**: V1.0  
**创建**: 2026-03-02  
**定位**: AHL核心竞争力的数据底座顶层设计  

**框架核心**:
```
精准匹配 = Σ(通用维度 × 特异化维度 × 实时动态系数)
护城河 = 时间积累 × 双边网络效应 × 数据飞轮
```

**维度体系**:
| 层级 | B端(供给侧) | C端(需求侧) | 匹配机制 |
|------|------------|------------|---------|
| 通用基底 | 物理属性/设施/价格/口碑 | 人口统计/设备渠道 | 硬性约束过滤 |
| 特异化核心 | 场景适配/内容表达/动态运营/关系网络 | 出行意图/需求层次/偏好指纹 | 相似度计算 |
| 深度飞轮 | B端AGENT沉淀/C-B反馈 | 行为轨迹/实时动态/AGENT交互 | 反馈闭环优化 |

**项目关联**:
- **06-AHL-去中心化旅行平台**: C端AI管家 + B端AI运营官 + 匹配引擎 (P0核心)
- **04-单体酒店AI计划**: AI获客工具 (场景维度应用)
- **01-自媒体计划**: 内容选题 (用户画像洞察)

**实施阶段**:
- Phase 1 (0-3月): 基础设施层 - 通用维度数据基线
- Phase 2 (3-6月): 场景层 - 7大场景适配能力
- Phase 3 (6-12月): 智能层 - 个性化精准匹配
- Phase 4 (12月+): 飞轮层 - 数据自运转
