## 🎯 使用指南

### P&P vs SOP 区别

| 维度 | P&P (政策与程序) | SOP (标准操作程序) |
|------|------------------|-------------------|
| **定位** | 做什么 (What) | 怎么做 (How) |
| **层级** | 战略/管理层面 | 执行/操作层面 |
| **内容** | 原则、规则、权限 | 步骤、方法、标准 |
| **更新** | 较少变动 | 定期优化 |
| **示例** | "所有预订需收取首晚房费担保" | "预订录入5步骤：1.查询房态→2.录入信息→3.确认价格→4.收取担保→5.发送确认" |

## 📊 内容规范

### P&P编写规范

**标准结构**:
1. **目的** - 为什么制定此政策
2. **适用范围** - 适用于谁/什么场景
3. **政策内容** - 具体规定
4. **程序** - 执行流程
5. **职责** - 谁负责
6. **相关文件** - 关联文档
7. **修订记录** - 版本历史

### SOP编写规范

**标准结构**:
1. **目的** - 此SOP的目标
2. **适用范围** - 适用于谁
3. **所需工具/材料** - 需要什么
4. **操作步骤** - 详细步骤（编号）
5. **质量标准** - 完成标准
6. **常见问题** - FAQ
7. **流程图** - 可视化流程

## ⚠️ 重要声明

1. **机密性**: 本体系为内部文件，不得外传
2. **合规性**: 所有政策需符合当地法律法规
3. **执行力**: P&P和SOP是运营底线，必须严格执行
4. **持续改进**: 鼓励员工提出优化建议
