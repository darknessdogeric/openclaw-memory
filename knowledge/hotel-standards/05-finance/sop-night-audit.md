# 夜审SOP (Night Audit SOP)

> **文档编号**: FI-SOP-001  
> **版本**: v1.0  
> **部门**: 财务部  
> **生效日期**: 2024-01-01

---

## 1. 目的

确保每日营业收入准确入账，账务平衡，为管理层提供准确的经营数据。

## 2. 适用范围

适用于夜班审计员每日夜审工作。

## 3. 工作时间

**时段**: 23:00 - 次日08:00

## 4. 所需工具/材料

- PMS系统
- POS系统
- 夜审报表
- 计算器
- 账单/收据
- 交接班记录本

## 5. 操作步骤

### 步骤1: 准备工作 (23:00-23:30)

**系统检查**:
- [ ] 登录PMS系统
- [ ] 检查系统运行正常
- [ ] 备份数据

**准备工作**:
- [ ] 整理当日账单
- [ ] 准备报表模板
- [ ] 与前台交接

### 步骤2: 数据核对 (23:30-01:00)

**房态核对**:
1. 打印在住客人报表
2. 核对实际房态与系统
3. 检查异常房态（如DND但系统显示空房）
4. 确认续住/退房

**账目核对**:
1. 核对当日入账
   - 房费
   - 餐饮收入
   - 迷你吧
   - 其他收入
2. 核对付款方式
   - 现金
   - 信用卡
   - 挂账
   - 第三方支付

**POS核对**:
1. 打印POS报表
2. 核对各餐厅收入
3. 核对支付方式
4. 核对折扣/赠送

### 步骤3: 过房费 (01:00-01:30)

**操作程序**:
1. 确认所有当日入住已入账
2. 确认所有退房已结账
3. 执行过房费操作
4. 生成房费明细

**检查要点**:
- 房价正确
- 税费正确
- 折扣已审批
- 特殊房价有备注

### 步骤4: 账务平衡 (01:30-02:30)

**试算平衡**:
1. 借方总额 = 贷方总额
2. 现金应缴 = 现金实缴
3. 信用卡应收 = 信用卡实收
4. 挂账增加 = 挂账减少 + 余额变动

**差异处理**:
- 小额差异(<¥10)：记录，次日调查
- 大额差异：立即调查，报告财务经理

### 步骤5: 报表生成 (02:30-04:00)

**必出报表**:

| 报表名称 | 内容 | 报送 |
|----------|------|------|
| 日报表 | 当日收入汇总 | GM、财务总监 |
| 房务报表 | 入住率、ADR、RevPAR | GM、销售总监 |
| 餐饮报表 | 各餐厅收入 | 餐饮总监 |
| 收款报表 | 各渠道收款 | 财务经理 |
| 应收账款 | 挂账明细 | 财务经理 |
| 在住报表 | 当日在住客人 | 前台 |
| 离店报表 | 次日预计离店 | 前台 |

**报表核对**:
- 数据准确
- 格式正确
- 及时发送

### 步骤6: 现金处理 (04:00-05:00)

**现金清点**:
1. 清点各收银点现金
2. 核对缴款单
3. 填写现金日记账
4. 封包存入保险箱

**备用金**:
- 补足前台备用金
- 记录补款

### 步骤7: 系统夜审 (05:00-05:30)

**夜审操作**:
1. 确认所有数据录入完成
2. 执行系统夜审
3. 生成审计报告
4. 关闭当日账期
5. 开启新账期

**注意**: 夜审后当日数据不可修改

### 步骤8: 交接与收尾 (05:30-08:00)

**工作交接**:
1. 填写交接班记录
2. 特殊事项说明
3. 待办事项移交
4. 与早班交接

**文件归档**:
- 报表存档
- 账单整理
- 记录归档

**系统退出**:
- 注销登录
- 关闭设备

## 6. 质量标准

| 指标 | 标准 |
|------|------|
| 数据准确率 | 100% |
| 报表及时性 | 08:00前发送 |
| 现金差错 | 0 |
| 系统故障响应 | <15分钟 |

## 7. 常见问题

**Q: 发现房费未入账怎么办？**
A: 联系前台核实，补入账，记录原因。

**Q: 系统夜审失败怎么办？**
A: 检查未平账目，处理差异，重新夜审。必要时联系IT支持。

**Q: 报表数据异常怎么办？**
A: 追溯数据来源，查明原因，修正后重新生成报表。

## 8. 安全事项

1. **数据安全**: 定期备份，密码保密
2. **现金安全**: 及时缴存，双人清点
3. **信息安全**: 报表不外泄，敏感数据加密
4. **人身安全**: 夜间注意安全，紧急情况按预案处理
